# 5.9 版本介绍

本文档说明 **Monopoly Deal Game** 课设代码在「5.9 版本」上下文下的**当日进度**与**全包全类方法职责**（尽量细到「谁调用、补哪块逻辑」）。  
代码中「日志记录」对应包名：`com.example.monopoly_deal_game.journal`，入口类 `Banben59Jieshao` 可打印本文件路径或全文。

## 一、今日进度汇报（工作摘要）

1. **领域模型（model）**  
   - 补齐 **`MonopolyDealOfficialDeck`**：按官方常见拆表生成约 110 张牌；开局由 **`GameLogic.initGame`** 再 **滤除 4 张 `RuleCard`**，实际进入牌库约 106 张可玩牌。  
   - **`Property`** 支持 **房屋/旅馆** 牌叠附在成套上，拆套时 **`takeAllBuildings`** 把房牌送去弃牌堆（避免「半套还挂房」的非法状态）。  
   - **`GameState`** 增加 **`doubleNextRent`**，给「加倍租金」行动牌与下一张租金结算衔接。

2. **规则与效果（logic）**  
   - **`CardEffectExecutor`**：实现钞票入账、物业落桌、租金、规则卡弃置、各类行动牌（摸牌、收债、生日、偷换、夺套、建房店等）；**目标玩家**在多人局暂取 **列表中第一个非当前玩家** 作为自动补位（给联机/之后 UI 选人留钩子）。  
   - **`PaymentService`**：支付时先 **银行区**（大额现金优先），再 **场上物业牌按面值抵**；缺的「补位」由 **下一张最小面值物业牌** 顶上，用于满足「收租/生日/收债」等扣款。  
   - **`RentCalculator` / `PropertyPlayHelper` / `TurnManager` / `CardManager`** 与 **`GameLogic.playCard`** 串成完整出牌管线。

3. **界面（view + controller）**  
   - **`GameplayViewCoordinator`**：除手牌外，刷新 **对手摘要条** 与 **己方桌面**（成套文字 + 缩小 `CardView` 缩略图 + 建筑缩略图），让打牌效果可见。  
   - **`AbstractGameplayScreenController`**：`playCard` 失败时 **`IllegalStateException` → feedbackLabel**；**`refreshSessionUi`** 给自动演示/脚本刷新 UI。

4. **自动演示（demo.autorun）**  
   - **`DemoGameplayApplication`**：启动后延时 **自动摸牌**、再 **自动连打** 手牌首张若干次，控制台打印快照，便于不写集成测试也能「看一眼」。

5. **已知简化**  
   - 未实现：Just Say No 的「对家打出瞬间插队」、弃牌阶段 UI、多人目标点选。  
   - 万能物业参与租金在部分边界上做了简化（杂色桌、全色万能收租取最大可收一档）。
