Package: com.example.monopoly_deal_game.model
================================================================================
[Included classes]
  - Property            A group of same-color/compatible {@link PropertyCard} lists in a property area (full set, monopoly, and rent; formerly PropertySet, merged into this class).
  - Player              Player (unified for human/bot).
  - ComputerPlayer      Bot, holding a BotPolicy (from the ai package).
  (The card family is in subpackage model.cards: Card, PropertyCard, BankCard, ActionCard, RentCard, CardColor, CardType)
  (The game session aggregate root and session state are in game.model: GameSession, GameState)
  (Also see package-info.java.)

[Interfaces]
  - No standalone interface file (Card is an abstract class, located in model.cards).

[What needs to be implemented]
  - Requirement 1: Deal 5 starting cards, draw pile/discard pile references (see game.model.GameSession + logic.CardManager).
  - Requirement 4-5: Bank area, property area grouping, Wild/dual-color toggle fields.
  - Requirement 16: Data structures needed for full set determination.
  - persistence: Session must be serializable (do not include JavaFX Node).

[Relationships with other packages]
  - ← logic: GameLogic, CardManager, CardEffectExecutor read/write game.model.GameSession.
  - ← persistence: SaveGameService serializes GameSession.
  - ← network: State broadcast by the host should be restorable to Session.
  - → ai: ComputerPlayer references BotPolicy (strategy interface in the ai package).
